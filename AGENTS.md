# @marianmeres/design-tokens — Agent Guide

## Quick Reference

- **Stack**: Deno / TypeScript
- **Run**: `deno task test` | **Watch**: `deno task test:watch`
- **Build (npm)**: `deno task npm:build`
- **Build (CSS)**: `deno task css:build` (generates pre-built CSS files; accepts `CSS_PREFIX` env var, default `"stuic-"`)
- **Build (themes barrel)**: `deno task themes:build` (regenerates `src/themes/mod.ts`)
- **Build (preview)**: `deno task preview:build`
- **Publish**: `deno task publish` (JSR + npm)
- **Format**: `deno fmt` (tabs, 90 chars, indent 4)

## What This Package Does

CSS custom property generator for design token systems. Takes a structured
`ThemeSchema` (light + optional dark mode) and produces CSS `--custom-property`
declarations with automatic hover/active state derivation via `color-mix()`.

Entry points:

- `.` — core generation functions, types, color palette
- `./reboot` — Bootstrap Reboot `--bs-*` variable bridge
- `./themes` — 43 pre-built theme definitions + `themeNames` constant + `bundledThemes` / `bundledThemeNames` / `getBundledTheme` runtime helpers
- `./css/*.css` — pre-built CSS files (**npm only** — JSR does not support directory exports)

## Project Structure

```
/src
  mod.ts              — main entry: re-exports functions, types, colors
  types.ts            — core type definitions (ThemeSchema, TokenSchema, etc.)
  generate.ts         — token generation + CSS output (generateCssTokens, toCssString, generateThemeCss)
  utils.ts            — hex color parsing (hexToRgb, hexToRgbTriplet)
  colors.ts           — Tailwind color palette (v3 + v4.2 additions: taupe, mauve, mist, olive), as const
  reboot-bridge.ts    — Bootstrap Reboot --bs-* variable mapping
  /themes
    mod.ts            — auto-generated barrel + runtime registry (via scripts/build-themes-mod.ts)
    *.ts              — individual theme files (default export: ThemeSchema)
/css                  — pre-built CSS files (generated by scripts/build-css.ts; filenames = kebab-case theme names)
/tests                — test files (deno test --allow-read)
/scripts
  build-npm.ts        — npm package build (also copies css/ into dist)
  build-css.ts        — generates css/*.css from all themes; honors CSS_PREFIX env var
  build-themes-mod.ts — auto-generates src/themes/mod.ts + inline registry helpers
/theme-preview-builder — HTML preview tool
/mcp.ts               — MCP tool definitions (generate-theme-css, preview-bundled-theme)
```

## Public API Surface

### Functions (from `./src/mod.ts`)

| Function                                      | Purpose                                                                                             |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| `generateCssTokens(schema, prefix, options?)` | Generate token key-value pairs. Third arg: `GenerateOptions` object or `"light"`/`"dark"` shorthand |
| `toCssString(tokens, selector?)`              | Convert tokens to a CSS declaration block (groups by base color name)                               |
| `generateThemeCss(schema, prefix, options?)`  | Generate complete CSS for light + optional dark mode; supports `cssLayer` wrapping                  |
| `hexToRgb(hex)`                               | Parse hex to `[r, g, b]` tuple                                                                      |
| `hexToRgbTriplet(hex)`                        | Convert hex to `"r, g, b"` string                                                                   |

### Functions (from `./reboot`)

| Function                                      | Purpose                                                                                                                                             |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `generateRebootBridge(tokens, prefix)`        | Map tokens to `--bs-*` variables. `-rgb` companions only emitted for literal hex values — auto-derived `color-mix()` values cannot produce triplets |
| `generateThemedCss(schema, prefix, options?)` | Combined tokens + reboot bridge CSS                                                                                                                 |

### Functions (from `./themes`)

| Function                | Purpose                                                                     |
| ----------------------- | --------------------------------------------------------------------------- |
| `getBundledTheme(name)` | Look up bundled theme by camelCase name; returns `ThemeSchema \| undefined` |

## Critical Conventions

1. All public functions must have JSDoc and explicit return types (Deno/JSR requirement)
2. Token naming: `--{prefix}color-{key}`, `--{prefix}color-{key}-hover`, etc.
3. Prefix is auto-normalized — `"my"` and `"my-"` both produce `--my-color-*`. Empty string produces unprefixed.
4. Hover/active auto-derived via `color-mix(in oklch, ...)` — light darkens with black, dark lightens with white. Disable with `deriveStates: false`.
5. Surface-intent tokens derived via `color-mix(in srgb, ...)` from intent + background. Foreground contrast defaults to 50% — tune via `surfaceForegroundContrast`.
6. Theme files use `export default` with `ThemeSchema` type annotation. All bundled themes include an optional `surface-1` paired role (extra elevation).
7. Formatting: tabs, 90-char line width, 4-space indent width (`deno fmt`)
8. MCP handlers must return `Promise<string>` — framework contract from `jsr:@marianmeres/mcp-server/types`.

## Adding a New Theme

1. Create `src/themes/{name}.ts` with `export default` of `ThemeSchema`
2. Run `deno task themes:build` to regenerate `src/themes/mod.ts` (barrel + registry)
3. Run `deno task css:build` to regenerate CSS files
4. Run `deno task preview:build` to rebuild the preview tool
5. If the theme uses new color families, update `COLOR_FAMILIES` in `theme-preview-builder/index.html`

## Known Limitations

- **JSR does not support directory exports** → pre-built CSS is npm-only.
- **Reboot `-rgb` companions** are omitted for tokens whose values are `color-mix()` expressions (auto-derived hovers). Provide explicit hex `hover` / `active` if you need full alpha-blend support through Reboot's `rgba(var(--bs-…-rgb), α)` rules.
